Setting Permissions for Snofyre Data

This section describes the process for setting up access permissions for the Snofyre data on the local MySQL server using the MySQL workbench.

  1. In the home screen of MySQL Workbench, double click on snofyre@localhost under the Server Adminstration as shown below.

  2. In the screen that opens, click on Accounts tab as shown in the screen below.

  3. In the screen that appears, click on the Schema Privileges tab as shown below.

  4. In the screen that appears, select snofyre user from the list of users under the Users list as shown below. Then click the Add Entry button, as shown below.

  5. In the dialog that opens:
    1. Select the Selected host option from under Host as shown below.
    2. From the dropdown list beside Selected host select localhost option as shown below.
    3. Select Selected schema from under Schema.
    4. In the list to the right, select fakedata2.
    5. Click the Ok button.

  6. In the same screen, select fakedata2 from under Select a user and pick privileges... and click Select ALL button from under Object Rights as shown below. Click the Save Changes button as shown below.

    MySQL Workbench should have now configured the appropriate permission for snofyre user to access Snofyre data.
