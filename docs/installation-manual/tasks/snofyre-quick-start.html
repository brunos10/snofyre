Snofyre Quick Start

The section describes the steps for running Snofyre after successful installation. The directory structure of the Installation Folder is shown below. The folder called bin contains all the executables for running Snofyre. The folder docs contains the API java docs and the Snofyre User Manual. The Uninstaller folder contains an executable that uninstalls Snofyre from the client machine. The folder queries contains the test cases that need to be executed for testing Snofyre.

  1. Navigate to the Installation Folder on your machine.
  2. Navigate to the bin sub folder, under the Installation Folder.
  3. You can launch Snofyre in your client in the following ways:
    • On the Windows platform , Snofyre can be also launched using the Snofyre.exe file located in the bin sub folder of the Installation Folder. This is typically done by double clicking the Snofyre.exe file.
      CAUTION:
      If you are using Windows Vista or 7, you need to right click on the Snofyre.exe executable located in the INSTALLATION_FOLDER/bin and select Run as Adminstrator to launch Snofyre correctly. The run.bat file will not work on these versions of Windows!
    • On the Windows XP and earlier, Snofyre can be also launched using the run.bat file located in the bin sub folder of the Installation Folder. This is typically done by double clicking the run.bat file.
      CAUTION:
      If you are using Windows Vista or 7, the run.bat file will not work on these versions of Windows!
    • Mac OS X users (10.5+) can launch Snofyre using Snofyre.app located in Installation Folder/bin. If it not present, you can download it by following this link.
    • On the *nix platform, launch Snofyre using the run.sh script, located in the bin sub folder of the Installation Folder.
    This starts up and displays Snofyre window on the screen as shown below.
    Attention: The first time Snofyre is run on the client machine, the application window is initialised as fairly small window.The user needs to resize the window. Snofyre will from then remember the user specified window size on each restart.
