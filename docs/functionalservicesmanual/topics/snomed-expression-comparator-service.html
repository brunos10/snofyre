SNOMED CT Expression Comparator Service

This module provides services for comparing SNOMED CT concepts and expressions.

What is it?

This module provides services for comparing SNOMED CT concepts and expressions.

Why do you need it?

One of the most commonly used feature in a SNOMED CT enabled application supporting querying is checking if a given expression A is a type of another given expression B. The comparision can also be performed against concepts. The expression comparision module provides a service that allows comparison of SNOMED CT concepts or expressions provided by SNOMED CT Object Model or SNOMED CT Expression Model modules respectively.

How do you use it?

The following figure illustrates the main classes implemented in this module. These classes implement common features needed to compare SNOMED CT Object Model or or SNOMED CT Expression Model classes.

The most important class in this model is ExpressionComparator. These classes provides the usual methods for comparing Expressions or SnomedConcepts. The following is a list of some of the commonly used methods in the above classes.

Important Classes and Methods

ExpressionComparator
Class Name Method Name
getSubsumptionRelation returns the Subsumption_Relation between two given Expressions or SnomedConcepts.
isSubsumedBy returns the boolean value for subsumption if the left hand side SnomedConcept Expression is subsumed by the right hand side SnomedConcept or Expression.
The class also provides methods for performing the comparing SnomedConcepts, by only providing the corresponding concept ids.

Gotchas

Important: ExpressionComparator checkes the UUIDs assigned to Expressions for equality, as a means for equivalence testing. In its default state, since UUIDs are assigned and managed by Snofyre automatically, you do not need to worry about this. But this is important to remember if you are manually assigning UUIDs.
