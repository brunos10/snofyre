SNOMED CT Normaliser Service

This module provides services for generating the long and short normal forms of a given SnomedConcept or Expression.

What is it?

This module provides services for generating the long and short normal forms of a given SnomedConcept or Expression.

Why do you need it?

While it is possible to compare two given SnomedConcepts for subsumption or equivalence using the Transitive Closure Table, this method can be be applied to comparing Expressions. The IHTSDO's approved methodology for comparing Expressions is to compare their Normal Forms. This requires the ability to convert a given SnomedConcept or Expression into its normal form. The generation of a normal form, involves applying IHTSDO approved heuristics recursively. This provides a service that implements these heuristics, making it easier for the developer to generate normal forms by calling the service.

How do you use it?

The most important class in this model is NormalFormGenerator. This classes provides the methods for for generating the long and short normal forms of a given SnomedConcept or Expression.

Important Classes and Methods

NormalFormGenerator
Table 1. Methods in NormalFormGenerator
Method Name Description
getLongNormalFormExpression returns the long normal form of a given SnomedConcept or Expression.
getShortNormalFormExpression returns the short normal form of a given SnomedConcept or Expression.
The class also provides methods for generating normal forms for SnomedConcepts, by only providing the corresponding concept id.

Gotchas

Important: NormalFormGenerator implements a method getLongNormalFormExpressionForRendering, which is an utility that is used to constrain and generate post-coordinated expression creation at the user interface. You are advise not to use this method because it is likely to be deprecated in future releases.
