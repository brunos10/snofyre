SNOMED CT Modules and Services

This section provides an overview of the SNOMED CT related functionality in Snofyre. While the following sections describe the individual modules (or services) in greater detail, this section describes how these modules are related to each other. The modular, service oriented architecture of Snofyre allows for modules to be linked, which can be visualised as a stack of modules built on top of each other, as shown in... insert figure here

Figure 1. Overview of SNOMED CT related services in Snofyre
At the bottom of the stack is the representation of SNOMED CT content (concepts, relationships, descriptions) provided by the SNOMED CT Object Model module. This module is an integral part of all other modules, which add more functionality on top of this module. For example, the SNOMED CT Expression Model uses the SNOMED CT Object Model module to represent SNOMED CT post-coordinated expressions. The modules at the top of the stack reuse the modules lower down, adding more functionality.
Tip: An overview of the components is presented below. A detailed description of the individual components and how to use them is presented in the Snofyre Functional Services Manual.
Table 1. SNOMED CT Service Components
Component Summary Description

This module provides services for generating the long and short normal forms of a given SnomedConcept or Expression.

This module provides services for returning parents, children, ancestors descendants and proximal primitives for a given SnomedConcept.
