Services Overview

This section descibes how the various service components in Snofyre are organised. It also introduces to some of the key vocabulary and that will be used later on in this guide.

Since Snofyre is based on functional architecture, it is composed of components that provide different functionalities. Each of these individual components that provides a specific application functionality or feature is referred to as a Module. Some of this vocabulary is borrowed from OSGi1, the modularity framework that Snofyre uses behind the scenes. A collection of modules that implement related software functionality, to satisfy a functional requirement is called a Service. In OSGi speak, a Snofyre module and service are equivalent to an OSGi bundle and service respectively.

The various modules (and services) that are implemented in Snofyre can be grouped into three broad categories:
  • SNOMED CT Services
  • Search Services
  • Query Services
Figure 1. Snofyre Service Layer Overview
SNOMED CT Services (and modules) implement functionality for representing, storing and performing common operations on SNOMED CT content (concepts, relationships and descriptions). On the other hand Query Services (and modules) deal with representing functionality for supporting querying of SNOMED CT encoded data (not SNOMED CT content). Most of the query related functionality has origins in the primary motivation of Snofyre, explained earlier. Conceptually it is useful to separate Search Services (and modules) from the rest of SNOMED CT Services because search functionality in Snofyre is not based on traditional database search. It uses Lucene to provide lexical search functionality.
Since Snofyre uses a number of number of open source frameworks to implement some of its functionality, the reader might find it useful to understand concepts related to these underlying frameworks. The following is a list of concepts that the might be of interest.
  • Dependency Injection (Spring , Spring DM)
  • Modularity (OSGi)
  • Object Relational Mapping (Hibernate)
  • Lucene
  • Tokenisation in Search
  • SNOMED CT Modules and Services
  • Snofyre Query Modules and Services
  • Search Services and Modules
