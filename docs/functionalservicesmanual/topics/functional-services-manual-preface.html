Snofyre Functional Services Manual - Draft

How this manual is organised

Motivation

Snofyre is a by-product of work undertaken by NHS Connecting for Health to demonstrate querying of SNOMED CT encoded data. Snofyre implements a number of SNOMED CT features as modular services, which many SNOMED CT implementers might be interested in. Snofyre code base and related dcoumentation is being made available under an open source license to enable implementers to use Snofyre as a starter for building their own SNOMED CT enabled systems. We hope that implementers use Snofyre's code as example code for implementing their own SNOMED CT functionality. Snofyre does not come with any warranty. Please refer to appropriate section of the license.

Content Organisation

The contents of this document are organised as module descriptions. Each module description is organised as follows:
  • Module Name
  • What is it?
  • Why do you need it?
  • How do you use it?
    • UML class diagrams (where appropriate)
    • Important classes, methods and configuration
  • Where to from here?
    • Links to appropriate packages
    • Examples if applicable
  • Gotchas (things to watch out for)!
  • Dependencies

Conventions Used

The following are some of the conventions used in this document:
  • Sections that have some importance are hilighted as :
    Important: Some important note here!
  • Sections that are noteworthy are hilighted as :
    Note: Some noteworthy information here!
  • Filepaths are shown as path/here/
  • Code is shown in typeface like this : 
    
// declaring some variable
 private String name;
 private int counter = 0;
