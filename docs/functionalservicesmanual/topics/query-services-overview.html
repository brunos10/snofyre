Snofyre Query Modules and Services

This section provides an overview of the query related functionality in Snofyre. While the following sections describe the individual modules (or services) in greater detail, this section describes how these modules are related to each other. The modular, service oriented architecture of Snofyre allows for modules to be linked, which can be visualised as a stack of modules built on top of each other, as shown below

Figure 1. Overview of Query related services in Snofyre
At the bottom of the stack is the representation of an abstract query formalism related classes, provided by the Snofyre Query Model module. This module is an integral part of all other modules, which add more functionality on top of this module. For example, the Snofyre Query Transformer Service generatse appropriate SQL statements based on the logical operators specified in the Snofyre Query Model module. In general, the modules at the top of the stack reuse the modules lower down, adding more functionality.
Tip: An overview of the components is presented below. A detailed description of the individual components and how to use them is presented in the Snofyre Functional Services Manual.
Table 1. Query Service Components
Component Summary Description
Snofyre Query Model

This module is a representation of the Snofyre's underlying query formalism as Java classes.
Snofyre Query Transformer Service

This module provides the methods that transform the a given QueryExpression into its corresponding SQL statements.
Snofyre Query DAO Service

This module provides the methods for storing and retrieving QueryExpressions stored in database.
Snofyre Data Generator Service

This module provides the methods for creating plausible clinical data containing pre and post-coordinated expressions.
