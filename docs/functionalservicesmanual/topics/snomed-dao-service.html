SNOMED CT Data Access Module (DAO Module)

This module provides services for accessing SNOMED CT content stored in a relational database.

What is it?

This module provides services for accessing SNOMED CT content stored in a relational database.

Why do you need it?

The data access services provide a level of abstraction, insulating the developer from the underlying schema of the relational database where SNOMED CT content is stored. The data access module returns the Java class representation provided SNOMED CT Object Model module.

How do you use it?

The following figure illustrates the main classes implemented in this module. These classes implement common features needed to return SNOMED CT Object Module classes.

Figure 1. Class model for DAO Module

The most important classes in this model are the TerminologyConceptDAO and SnomedExpressionDAO. These classes implement the usual methods for retrieving, saving and deleting SnomedConcepts and Expressions. The following is a list of some of the commonly used methods in the above classes.

Important Classes and Methods

TerminologyConceptDAO
Table 1. Methods in TerminologyConceptDAO
Method Name Description
getTerminologyConcept returns the SnomedConcept for the given concept id.
getCurrentConceptForConcept returns the appropriate current SnomedConcept for a given inactive SnomedConcept.
The class also provides methods for returning a collection of concepts based on a search terms. Please refer to the Javadocs provided for a complete list of methods.
Table 2. Methods in SnomedExpressionDAO
Method Name Description
saveExpression save the given Expression to the database.
find returns a matching Expression when a UUID is passed.
This class implements other utility methods for deleting expressions and returing all expressions in database. Please refer to the Javadocs provided for a complete list of methods.
Table 3. Methods in TerminologyConceptUtils
Method Name Description
getConceptForID a static utility method that returns the SnomedConcept for the given concept id.

Gotchas

Note: All instances of concepts returned by TerminologyConceptDAO can be safely be cast to SnomedConcept. This might change if an different concrete implementation to support other terminologies is implemented.
Important: We do not recommend calling the getAllConceptsInTerminology method in TerminologyConceptDAO.
Important: The current implementation of TerminologyConceptDAO focusses on reead-only call for retrieving SNOMED CT content. The methods for saving content are not implemented!
