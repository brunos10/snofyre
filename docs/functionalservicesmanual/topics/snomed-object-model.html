SNOMED CT Object Model Module

This module is a representation of relevant SNOMED CT content as Java classes.

What is it?

This module is a representation of relevant SNOMED CT content as Java classes.

Why do you need it?

The object model representation allows the programmer to represent and manipulate SNOMED CT content as pure old Java objects. If you intend to do anything more than a few calls to look at SNOMED CT content, it makes a lot of sense to work with SNOMED CT content as Java objects.

How do you use it?

The following figure illustrates the main classes implemented in this module. These represent the most commonly used aspects of SNOMED CT's content.

The most important classes in this module are the SnomedConcept, SnomedDescription, SnomedRelationship and SnomedRoleGroup. These classes represent the corresponding notions as described in the IHTSDO's Technical Implementation Guidance document. The following is a list of some of the commonly used methods in the above classes.

Important Classes and Methods

SnomedConcept
Table 1. Methods in SnomedConcept
Method Name Description
getConceptId returns the concept id of this concept
getDescriptions gets all the SnomedDescriptions on this concept
getFullySpecifiedName gets the fully specified name of this concept
getSourceRelationships gets the SnomedRelationships for this concept is the source
getTargetRelationships gets the SnomedRelationships for this concept is the target
getRoleGroups gets the SnomedRoleGroups for this concept
getParents gets the SnomedConcepts that are parents of this concept
getChildren gets the SnomedConcepts that are children of this concept.
The class also provides methods for getting descendants, ancestors, corresponding CTV3 ID, etc. Please refer to the Javadocs provided for a complete list of methods.
SnomedDescription
Table 2. Methods in SnomedDescription
Method Name Description
getTerm gets the textual term associated with this description
getConceptId gets the id of the SnomedConcept, this description is associated with.
This class implements other methods for getting the language, description type, etc. Please refer to the Javadocs provided for a complete list of methods.
SnomedRelationship
Table 3. Methods in SnomedRelationship
Method Name Description
getRelationshipType gets the relationship type id as specified in SNOMED CT.
getRefinability returns the value of refinability (one of mandatory, not-refinable and optional).
There are other methods for getting the source and target SnomedConcepts, relationship id, etc. Please refer to the Javadocs provided for a complete list of methods.

Gotchas

Note: The SnomedConcept class only has methods to return the ids for the descendants and ancestors of a given concept. Returning actual SnomedConcepts causes excessive memory usage because of the large object graphs this causes.
Note: The current implementation does not implement support for managing Subsets.
Attention: The current implementation only has minimal support for tracking historical relationships.
Remember: The current implementation is based on the Release Format 1 (RF1) data structures. It does not implement implement Release Format 2 specific features.
