Snofyre Data Generator Service

This module provides the methods for creating plausible clinical data containing pre and post-coordinated expressions.

What is it?

This module provides the methods for creating plausible clinical data containing pre and post-coordinated expressions.

Why do you need it?

One of the principal challenges in creating clinical applications, is access to clinical data. For various reasons, it is difficult for a developer to test and build a system against clinical data. This becomes doubly difficult when the clinical data needs to be SNOMED CT encoded, because not many SNOMED CT based systems exist (certainly not ones that generate post-coordinated data). Snofyre Data Generator Service solves the problem of access to clincical data, by generating plausible (realistic) clincial data that contains pre and post-coordinated expressions.

How do you use it?

The following figure illustrates the main classes implemented in this module.

Figure 1. Class Model for Query Objects

The most important classes in this module are DataGenerationEngine and RandomSubtypeGenerator. The DataGenerationEngine class represents the base methods implemented by the service. The service needs parameters that tell it what SNOMED CT expressions are contained in the generated data. So it uses a QueryStatement as a means of specifying the parameters. RandomSubtypeGenerator generates valid expressions that are used to populate the simulated records.

Gotchas

Tip: The DataGenerationStrategy determines if simulated data is added to new or existing records.
Important: The RandomSubtypeGenerator uses randomisation at multiple levels when creating subtypes for expressions. Currently not all of these randomisation parameters can not be pre-defined.
