SNOMED CT MRCM Service

This module provides services for retrieving sanctioned attributes for a given SnomedConcept based on the Machine Readable Concept Model.

What is it?

This module provides services for retrieving sanctioned attributes for a given SnomedConcept based on the Machine Readable Concept Model.

Why do you need it?

SNOMED CT provides the ability to create post-coordinated expressions at run time, by combining two or more existing SnomedConcepts. Since in the absense of any constraints on how post-coordinated expressions are created, it is possible to create non-sensical expressions. For example, in the absence of rules, it will be possible to create a post-coordinated expression that represents Pain with finding site Aspirin tablet. The Machine Readable Concept Model (MRCM) provides the rules that describe permissible (sanctioned) relationships between SnomedConcepts. For example, the MRCM does not sanction a relationship of type finding site between Pain and Aspirin tablet.

The MRCM service provides the calls that permit lookup of sanctioned attributes for a given SnomedConcept or concept type.

Gotchas

Important: MRCMService does not support cardinality related constraints and also does not manage any refinements that are inherited down the concept model. For example, it is currently not possible to use MRCMService to constrain a Situation with Explicit Context, to have either an associated finding or associated procedure, but not both. This feature might be implemented in a future release.
