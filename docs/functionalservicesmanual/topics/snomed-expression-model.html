SNOMED CT Expression Model Module

This module is a representation of relevant SNOMED CT post-coordinatd expressisons as Java classes.

What is it?

This module is a representation of relevant SNOMED CT post-coordinated expressions as Java classes.

Why do you need it?

The object model representation allows the programmer to represent and manipulate SNOMED CT post-coordinated epxressions as pure old Java objects. This allows the developer to transform post-coordinated expressions into different forms. Post-coordinated expressions are typically nested objects with 'expressions' contained inside other 'expressions'. This module provides the functionality for handling these nested structures.

How do you use it?

The following figure illustrates the main classes implemented in this module. These represent the most commonly needed features for using post-coordinated expressions.

The most important classes in this model are the Expression, CloseToUserExpression, NormalFormExpression and SituationExpression. These classes represent the corresponding notions as described in the IHTSDO's Technical Implementation Guidance document. The following is a list of some of the commonly used methods in the above classes.

Important Classes and Methods

Expression
Table 1. Methods in Expression
Method Name Description
getChildExpressions returns the child Expressions contained in this expression.
getParentExpressions returns the parent Expressions of this expression.
The class also provides methods for assigning identifiers and equivalent expressions. Please refer to the Javadocs provided for a complete list of methods.

Table 2. Methods in NormalFormExpression

This class implements other utility methods for returning canonical form, singleton focus concepts, etc. Please refer to the Javadocs provided for a complete list of methods.

SituationExpression
Table 4. Methods in SituationExpression
Method Name Description

Gotchas

Important: All instances of Expression use the canonical form to determine equality. This could be moved to a 'visitor' pattern in future releases.
Important: Note that using the getSingletonFocusConcept method on CloseToUserExpression or NormalFormExpression will force the expression to lose the rest of the focus concepts, if present. This is currently logged as a warning in the logs.
Note: Subclasses of SituationExpression validate objects passed to getFocusExpression or getFocusConcept. Please refer to the javadocs for more information.
