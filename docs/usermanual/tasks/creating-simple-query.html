Creating a Simple Query

This section describes how to create a new query in Snofyre.

  1. A new query can be created in any of the following ways:
    • Select File > New Query.
    • Click the New Query button on the toolbar.
    A stub entry for a new query is created in the Query Authoring Panel and this is reflected in the Query Rendering Panel. This will clear any query that has already been loaded in a previous step or an unsaved query.
    CAUTION:
    This action will result in the loss of any modificiations that the user might have made to an existing query, already loaded.
  2. Right click Get patients who have stub in the Query Authoring Panel. The same effect can be achieved by a combination of Crtl button and left click on the Windows platform and Cmd button and left click on the Mac platform. This results in appearance of the query creation pop up menu shown below.

  3. In the query creation pop up menu, click on Add Query Criterion as shown below.



    This displays a dialog as shown below.

  4. In this dialog, click on the Edit button as shown below.



    This displays a dialog as shown below.

  5. In this dialog, click on the Edit button as shown below.



    This brings up a search box in the dialog, as shown below.

  6. In the search box, type the text to search for, as shown below.

      Tip: You do not need to type in the entire word/phrase, to bring up results in Snofyre. You can use the first few characters of a term to bring up relevant matches. The example below shows this functionality for Myocardial Infarction.



    2. Press the Enterkey on your keyboard to return matches.
    Snofyre brings up matching SNOMED CT concepts as shown below.

  7. In this list, double click on the most appropriate result shown below.



    This closes the search box and displays the selected concept in the original dialog, as shown below.

  8. Dismiss this dialog, by closing it. This closes the dialog and takes us back to the query criterion dialog, shown below.

  9. Dismiss this dialog, by closing it. This closes the dialog and updates the query in the Query Authoring Panel and the Active Query Panel.
  10. It should now be possible to save or execute the newly created query.
