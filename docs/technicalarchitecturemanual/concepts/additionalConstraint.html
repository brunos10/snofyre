Additional Constraint

An Additional Constraint is a numerical or temporal criterion that can be used to refine the records that match a Query Component Expression.

Numerical Constraint

These constraints represent numerical values that can be used as boundaries on appropriate Query Criteria. These typically are useful for generating combinations of Observable Entities + value.

Query | Get all patients who have 'Hemoglobin Concentration' LESS THAN 10| uses a numerical constraint value of '10' to retrieve record entries with 'Hemoglobin Concentration' less than 10.
Temporal Constraint

Represent constraints that can be applied to the Sub Query Expression with respect to the current system time.

Important: A Temporal Constraint is not the same as a Temporal Operator. Temporal Operators (yet to be supported) will permit operators like AFTER, BEFORE, etc between two Query Expressions.
Query | Get all patients who have 'Hemoglobin Concentration' LESS THAN 10 WITHIN the last 6 months|
  • uses a numerical constraint value of '10' to retrieve record entries with 'Hemoglobin Concentration' less than 10
  • uses a temporal constraint WITHIN with a value specified for 6 months.
