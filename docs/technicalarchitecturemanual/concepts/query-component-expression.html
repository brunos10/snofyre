Query Component Expression

A Query Component Expression is an entity that is encapsulates a Query Term and necessary descriptions that qualify the record entries matched against the search token contained within the Query Term. A Query Component Expression is especially useful for defining the precise boundaries on a 'search token' in relation to the subsumption and concept descriptions that are features of SNOMED CT.

Included Query Criterion

A Query Component Expression always encapsulates a single 'search token' in the form of a Query Criterion to specify the record entries to be matched against. This Query Criterion instance is referred to as the Included Query Term since all record entries matching this term will be returned. The scope of records returned can be narrowed or broadened using Subsumption strategy of the query term.

Query | Get all patients who have 'Myocardial Infarction'| is a Sub Query Expression that contains a Query Term 'Myocardial Infarction'.

Exluded Query Criteria

Though the boundaries of a 'search token' can be defined using the Subsumption strategy property on the Query Criterion, there are cases where some 'search tokens' will need to be excluded from the concept hierarchy specified by the Subsumption Strategy. The Query Component Expression allows these excluded search tokens to be specified as 'Excluded Criteria' within the results matched by the 'search token' of the Included Query Criteria.
Note: A Query Component Expression can contain zero or more excluded query terms. Each of these Excluded Query Criteria can in turn specify their own Subsumption strategy.
Query | Get all patients who have 'Myocardial Infarction' EXCEPT 'Acute Q Wave Myocardial Infarction'| is a Sub Query Expression that contains:
  • Included Query Term : 'Myocardial Infarction'
  • Excluded Query Term : 'Acute Q Wave Myocardial Infarction'

Additional Constraints

Additional Constraints represent another way of refining the matches that are returned by a Query Component Expression. The combination of Subsumption strategy, excluded terms and additional criteria offers the user a generic but effective mechanism to specify the criteria for record retrieval.
