Data Generation Strategy

Data generation strategy refers to the selection process by which Snofyre chooses to add new record entries to existing patient records or to new patient records during data generation.

"The total number of clinical entries stored in Snofyre at any given point of time is a product of the number of patient records and the number of clinical entries in each patient record. " So, there are at least three different ways in which the total number of clinical entries in Snofyre's underlying database can be increased.
  • Increasing the absolute number of patient records
  • Increasing the number of clinical entries in each patient record.
  • Increasing numbers of both clinical entries and patient records.
Snofyre supports three flavours of Data generation strategies that roughly correspond to these notions.
Table 1. Flavours of Data Generation Strategy
Strategy Name Summary Description
ADD_NEW_ALWAYS Always increases the absolute number of patients by creating new patient records.
ALWAYS_APPEND Only ever adds new entries to existing patient records.
ADD_IF_NOT_EXISTS Only adds new patient records if specified entries number exceeds existing number of patients.
For example, if Snofyre's database already contains 100 record entries (100 patient records with 1 clincal entry each) and we chose to add 500 new record entries with different data generation strategies, here is what would happen:
Table 2. Flavours of Data Generation Strategy
Strategy Name Result
ADD_NEW_ALWAYS Result in a total of 600 patient records with 1 clinical entry in each patient record.
ALWAYS_APPEND Results in a total of 100 patient records with all new 500 entries randomly added to the patient records. So on an average each patient record might have about 6 clinical entries.
ADD_IF_NOT_EXISTS Results in the creation of 500 patient records, with 2 entries per record for all existing 100 records and 1 entry per record for each of the newly created 400 patient records.
