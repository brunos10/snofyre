Snofyre Information Model Module

This module provides contains a representation of all the Java classes that constitute the Information Model of Snofyre.

What is it?

This module provides contains a representation of all the Java classes that constitute the Information Model of Snofyre.

Why do you need it?

Most clinical queries invariably incorporate some aspect of the underlying information model (e.g. date of entry, value of a blood chemistry test, etc). In order to support some of these queries and also to create a sensible way to organise records, Snofyre implements a simplistic Information Model to store data generated by the Snofyre Data Generator Service.

How do you use it?

The Information Model classes used represent the usual clinical notions of findings, procedures, etc. However, Snofyre groups these concepts into Clincial Entities (the finding for example) and its corresponding mention in the clinical record (a Clinical Entry). While this is somewhat confusing for some, the author created this representation with an academic interest. The developer is not expected to use this as a starting point for their own implementations. In fact, the queries themselves are performed entirely against 'Clinical Entry' classes. Gotchas

Important: The recommended method for creating classes in the Information Model is to use the factory classes like ClinicalEntityFactory, ClinicalEntryFactory and EHRFactory.
Note: The following DAO implementations exist in the module; EhrDao and ClinicalEntryDAO. These are not directly used by Snofyre.
