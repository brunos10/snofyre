Snofyre Technical Architecture Manual - Draft

Query Entities and Relationships

The ability of the Snofyre to author and execute complex queries is dependent on the ability to combine different simpler query entities to create a more complex query. This section provides an overview of the different entities available in Snofyre.

Table 1. Query Entity Types
Entity Type Short Description
Query Expression

A Query Expression represents the abstract notion of an atomic entity that that is used to create a query. In fact, all query entities available within the Snofyre like Sub Query Expression, Union Expression and Intersection Expression are all types of a Query Expression. This high level abstraction allows for a query to be authored using any combination of sub query, union or intersection expressions.
Query Criterion

A Query Criterion is the SNOMED CT encoded 'token' which the user specifies as the criterion for retrieving clinical records/entries that match the 'token'. A Query Criterion can either be a SNOMED CT concept or a SNOMED CT encoded post-coordinated expression.

Query Component Expression

A Query Component Expression is an entity that is encapsulates a Query Term and necessary descriptions that qualify the record entries matched against the search token contained within the Query Term. A Query Component Expression is especially useful for defining the precise boundaries on a 'search token' in relation to the subsumption and concept descriptions that are features of SNOMED CT.

Intersection Expression

An Intersection Expression represents a collection of two or more Reporting Expressions joined using a logic AND operator (Intersection operator in Set theory).
Union Expression

An Union Expression represents a collection of two or more Query Expressions joined using a logic OR operator (Union operator in Set theory).
Query Statement

A entity that encapsulates one or more Query Expressions and contains sufficient logic to be executed against a data store. It is the equivalent of the logical transform of a user defined reporting query that is to be executed for data retrieval.
Additional Constraint

An Additional Constraint is a numerical or temporal criterion that can be used to refine the records that match a Query Component Expression.

Query Enity Relationships

This section provides an overview of the relationships between the different query entities. Since a Query Statement is an instance of Query Expression, it can contain other Query Expressions that in turn can contain other Reporting Expressions. This extensible model allows a Query Statement to be created using Query Component, Intersection and Union expressions as components. Each of these components can be refined using Additional Constraints.
