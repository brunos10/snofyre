Snofyre Technical Architecture Manual - Draft

Interpreting Results

Every succesful execution of a query results in the generation of a result set of record entries that fulfilled the criteria specified by the query. The result set so generated can be inspected visually using the Result View Panel of Snofyre.

Structure of Result Set

All result sets returned by the Snofyre are organised in a tabular structure with every row of the table corresponding to a record entry that satisfied the criteria specified by the query. Every row in this tabular structure has the following columns.

  • Patient Id : The id of the patient who satisfied all the criteria defined in the query.
  • Free Text Entry : The free text entry in the record that matched the search token defined in the sub queries of the query. As a rule, there will be one free text entry column for every necessary search token of every reporting expression contained in the query.
    Note: In this context, a 'necessary' search token in a reporting expression is defined by the logical operator specified over the expression. For example, if the reporting expression in question is an Union Expression, then there will be only one 'necessary' search token for all the reporting expressions contained inside the Union Expression since it is necessary for any one of the contained search tokens to be present. In the case of an Intersection Expression, every contained reporting expression will contribute a free text entry column since every search token in the contained expression needs to be present.

    For example, the Query | QueryCriterion1 AND (QueryCriterion2 OR QueryCriterion3)| will result in the generation of two free text entry columns. The rationale behind this is that all records that match satisfy this query will necessarily have the search token contained in QueryCriterion1 but can contain a search token contained in QueryCriterion2 or QueryCriterion3.

  • Concept Id : The corresponding concept Id (if present) in the record that matched the search token defined in the sub queries of the query. As with free text entry, there will be one concept id column for every necessary search token of every reporting expression contained in the query.
  • Entry Time : The corresponding entry time for the entry in the record that matched the search token defined in the sub queries of the query. As with free text entry, there will be one entry time column for every necessary search token of every reporting expression contained in the query.
Query Time
This is the actual time required to return result set after executing the query against the test data. The actual length of the query time is determined by :
  • The time required to tranform the query into its corresponding intermediate representation (SQL or other Java object) within Snofyre
  • The time required to execute the resultant intermediate form against the test data.
  • The time required to retrieve the result set returned by the executing the intermediate form against the test data.
    Note: Query Time displayed in Snofyre's graphical user interface might not reflect the actual time taken for the results to be displayed on the screen. Within the graphical user interface, additional time is required to render the returned result set for display.
