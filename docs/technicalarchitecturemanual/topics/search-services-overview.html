Search Services and Modules

This section provides an overview of search related functionality in Snofyre. While the following sections describe the individual modules (or services) in greater detail, this section describes how these modules are related to each other. The modular, service oriented architecture of Snofyre allows for modules to be linked, which can be visualised as a stack of modules built on top of each other, as shown in... insert figure here

Figure 1. Overview of Search services in Snofyre
At the bottom of the stack is the Search Service module, which implements lexical search against SNOMED CT content. This module is an integral part of all other modules, which add more functionality on top of this module. For example, the Search Panel module uses the Search Service module to present the search results in a List. The modules at the top of the stack reuse the modules lower down, adding more functionality. Most of these modules are dedicated to providing re-usable user interface components for searching and building expressions.
Tip: An overview of the components is presented below. A detailed description of the individual components and how to use them is presented in the Snofyre Functional Services Manual.
Table 1. Search Service Components
Component Summary Description
Search Service

This module provides services for searching SNOMED CT for matches against a given term (or phrase) or concept id.
Expression Builder A component that allows the user to create post-coordinated expressions
Search Panel A component with all the logic needed for presenting search results to the user in a JList.
