Information Model Entity Relationships

This section provides an overview of the different entities that are used internally by Snofyre's Information Model and the relationships between these entities.

The following is a short summary of the various entities used in Snofyre's Information Model.

Table 1. Information Model Entity Types
Entity Type Short Description
Clinical Entry

An interface that represents an entry asserted in the clinical record. It is guaranteed to have a clinical entry time and a patient ID associated with it. In practice, it also has a SNOMED concept ID (or a clinical expression) and a free text entry (the human readable entry corresponding to the concept id or clinical expression as asserted by the clinician).
Clinical Event

An interface that represents an event entry asserted in the clinical record. It is a special kind of Clinical Entry that has a Status associated with it. For this reason, it is guaranteed to have all the attributes of a Clinical Entry, but also have an additional attribute Status. The Status attribute describes if the clinical event has been planned, completed or aborted.
Anatomical Location Entity

An interface representation of an Anatomical Location or Body part. This usually corresponds to a SNOMED CT concept of type 'Body Structure'. It usually has an associated 'Laterality' which describes if the anatomical entity is on the left or right hand side of the body where appropriate.

Clinical Finding Entity
An interface representation of a Clinical Finding recorded in a clinical encounter as asserted in the clinical record. This usually corresponds to a SNOMED concept type of 'Clinical Finding', though it might also correspond to 'Situation with explicit concepts'. It is often associated with the attribute 'Finding Site' which is the Anatomical Location where the finding is located (when appropriate).
Note: Please note that a Clinical Finding Entity shares similarities with a Clinical Observation Entity. Please refer to Clinical Observation Entityfor information.
Clincial Observation Entity

An interface representation of an 'clinical observation' as asserted in the clinical record. It differs from a Clinical Finding entity which is also a 'clinical observatio' in that a Clinical Observation enity refers to the 'clinical observation' described as a SNOMED CT Observable Entity associated with a numerical value.
Investigation Entity

An interface representation of an Investigative procedure peformed on the patient as persisted in the clinical record. This usually corresponds to a SNOMED CT concept of type 'Procedure'. It is usually associated with a Target Site, an anatomical location which is the site at which the investigation has been targetted.
Intervention Entity

An interface representation of an Intervention that is carried out on the patient as asserted in the clinical record. This entity usually corresponds to a SNOMED CT concept of type 'Procedure'. It is usually associated with a 'Target Site' which is the Anatomical Location Entity at which the intervention has been targetted.
Medication Entity

An interface representation of a Medication entry in the patient record. It is guaranteed to have a of type "Product" associated with it and a time of administration. It will also usually have a dose associated with it.
